# Akamai CPS Certificate Management Script

A comprehensive Bash script for managing SSL/TLS certificates in Akamai's Certificate Provisioning System (CPS). This script automates the entire certificate lifecycle: enrollment creation, CSR retrieval, certificate issuance via DigiCert, upload to Akamai, and production deployment.

**Version:** 2.2.0

## Features

- **Automated Enrollment**: Create and manage CPS enrollments with full configuration support
- **CSR Management**: Automatic retrieval and storage of Certificate Signing Requests (RSA & ECDSA)
- **DigiCert Integration**: Seamless certificate issuance through DigiCert API
- **Intelligent Polling**: Automatically waits for CSRs and certificates to become available
- **Certificate Upload**: Upload certificates and trust chains directly to Akamai CPS
- **Warning Handling**: Automatically detect and acknowledge post-verification warnings
- **Production Deployment**: Full control over staging and production certificate deployment
- **Certificate Renewal**: Streamlined renewal workflow for existing enrollments
- **Configuration Management**: Save and load configurations for easy certificate renewal
- **Automated Mode**: Perfect for cron jobs with --auto flag (no interactive prompts)
- **Discovery Mode**: List and explore existing enrollments in Akamai

## Prerequisites

### System Requirements
- Linux/Unix environment (Ubuntu, CentOS, Debian, etc.)
- Bash 4.0 or higher
- Internet connectivity

### Required Tools (auto-installed by script)
- **Python3** and pip3 - For HTTPie installation
- **HTTPie** - For making API calls (installed via pipx)
- **httpie-edgegrid** - Akamai EdgeGrid authentication plugin
- **jq** - JSON query and processing utility
- **curl** - For DigiCert API calls (usually pre-installed)
- **openssl** - Optional, for certificate inspection (usually pre-installed)

### Akamai Credentials
- **EdgeGrid API Credentials** (from Akamai Control Center)
  - `host`
  - `client_secret`
  - `access_token`
  - `client_token`
- **Contract ID** (from Akamai Control Center)

### DigiCert Credentials
- **API Key** (from DigiCert MPKI)
- **Seat ID** (for certificate issuance)

## Quick Start

### Initialize a New Certificate Enrollment
```bash
./akamai-cps-script.sh --init
```
The script will interactively prompt for:
- Akamai EdgeGrid credentials
- Certificate storage location
- Contract ID
- CSR information (CN, organization, etc.)
- Network configuration
- Contact information

### Renew an Existing Certificate (Interactive)
```bash
./akamai-cps-script.sh --renew example.com
```
Before renewal, initiate the renewal in Akamai Control Center. The script will:
- Fetch the latest pending change
- Retrieve the CSR generated by Akamai
- Issue a new certificate
- Upload and deploy automatically

### Renew Using Configuration File
```bash
./akamai-cps-script.sh --renew-config ~/.akamai-cps/configs/example.com_enrollment_*.json
```

### List Saved Configurations
```bash
./akamai-cps-script.sh --list
```

### Discover Existing Enrollments
```bash
./akamai-cps-script.sh --discover
```

### Automated Mode (Cron-Friendly)
```bash
./akamai-cps-script.sh --renew example.com --auto
```
In automated mode, the script:
- Uses configuration values for all decisions
- Requires no interactive prompts
- Automatically submits to DigiCert
- Automatically uploads and deploys to production
- Perfect for scheduled renewal via cron jobs

## Workflow Overview

### Initialization Mode (`--init`)

```
1. Prerequisites Check
   └─ Verify/install Python3, pip3, HTTPie, jq, etc.

2. EdgeGrid Authentication
   └─ Collect and save Akamai API credentials

3. Storage Configuration
   └─ Set certificate storage directory

4. Contract Information
   └─ Specify Akamai contract ID

5. Enrollment Configuration
   └─ Define certificate type, CSR details, network settings

6. Contact Information
   └─ Admin and technical contact details

7. Organization Details
   └─ Organization name, address, contact info

8. Enrollment Submission
   └─ Create and submit enrollment to Akamai CPS

9. CSR Retrieval
   └─ Poll for CSR availability (automatic, ~10 sec intervals)
   └─ Extract RSA and ECDSA CSRs from response

10. Certificate Issuance
    └─ Select certificate type (RSA or ECDSA)
    └─ Submit CSR to DigiCert API
    └─ Wait for certificate issuance

11. Certificate Upload
    └─ Combine certificate with trust chain
    └─ Upload to Akamai CPS

12. Post-Verification Handling
    └─ Poll for and handle post-verification warnings
    └─ Acknowledge warnings if necessary

13. Production Deployment
    └─ Acknowledge change management
    └─ Monitor deployment status
    └─ Confirm production activation
```

### Renewal Mode (`--renew`)

```
1. Load Configuration
   └─ Retrieve saved certificate configuration

2. Check for Pending Changes
   └─ Poll Akamai for latest renewal change
   └─ Extract CSR generated by Akamai

3. Certificate Issuance
   └─ Submit CSR to DigiCert
   └─ Issue new certificate

4. Upload & Deployment
   └─ Upload certificate to existing enrollment
   └─ Handle warnings and deploy to production

5. Update Configuration
   └─ Update last_renewed timestamp
```

## Automated Deployment

### Automated Mode (`--auto` flag)

When using the `--auto` flag, all decisions are made automatically based on configuration values:

- **Certificate Type**: Uses RSA or ECDSA from config (default: RSA)
- **DigiCert Submission**: Automatically submits if enabled in config
- **Certificate Upload**: Automatically uploads if enabled in config
- **Warning Acknowledgement**: Automatically acknowledges warnings if enabled
- **Production Deployment**: Automatically deploys if enabled in config

### Example Cron Job

```bash
# Daily renewal check at 2 AM
0 2 * * * /opt/scripts/akamai-cps-script.sh --renew example.com --auto >> /var/log/akamai-renewal.log 2>&1

# Monthly renewal attempt on 1st day of month at 3 AM
0 3 1 * * /opt/scripts/akamai-cps-script.sh --renew example.com --auto >> /var/log/akamai-renewal.log 2>&1
```

```bash
# View renewal logs

grep -i "error\|failed" /var/log/akamai-renewal.log

ls -la ~/.akamai-cps/configs/ | grep example.com
```

## Certificate Management Workflows

### Adding a New Domain

1. Run the initialization script for the new domain
2. Complete the interactive setup
3. Configuration is automatically saved
4. Certificate is ready to renew annually

### Renewing a Certificate

1. **Initiate in Akamai Control Center**
   - Navigate to CPS
   - Find the enrollment for your certificate
   - Click "Renew"

2. **Run the renewal command**
   ```bash
   ./akamai-cps-script.sh --renew example.com --auto
   ```

3. **Monitor deployment** (optional)
   - Script will provide status updates
   - Certificate deploys to staging first, then production

### Viewing Certificate Details

Certificates and CSRs are stored in the configured storage directory:

```bash
# View all files for a domain
ls -la ~/akamai-certs/*example.com*

# Inspect certificate details
openssl x509 -in ~/akamai-certs/example.com_RSA_certificate.pem -noout -text

# Check CSR details
openssl req -in ~/akamai-certs/example.com_enrollment_12345_RSA.csr -noout -text
```

## Troubleshooting

### CSR Not Available After 5 Minutes

**Symptom**: Script times out waiting for CSR retrieval

**Solutions**:
1. Run the script again - Akamai may still be generating the CSR
2. Check enrollment status in Akamai Control Center
3. Verify your Akamai API credentials are correct
4. Ensure the enrollment ID is valid

```bash
./akamai-cps-script.sh --discover  # Check enrollment exists
```

### DigiCert Certificate Issuance Fails

**Symptom**: "Failed to submit certificate request to DigiCert"

**Solutions**:
1. Verify DigiCert API key is correct
2. Check seat ID configuration
3. Ensure CSR format is valid
4. Confirm API key has necessary permissions

### Certificate Upload Fails

**Symptom**: "Failed to upload certificate to Akamai"

**Solutions**:
1. Verify ICA certificate file exists and is readable
2. Check that certificate matches CSR
3. Verify Akamai API credentials
4. Ensure contract ID is correct

```bash
# Check ICA file
cat ~/.akamai-cps/configs/rudloff-ica.pem | head -5
```

### Post-Verification Warnings Not Appearing

**Symptom**: Script shows "No post-verification warnings detected" but deployment doesn't proceed

**Possible Causes**:
- Warnings may not be generated immediately
- Certificate may be automatically approved
- Change management may be required instead

**Solutions**:
- Allow time for warnings to generate
- Check change management status in next polling cycle
- Review Akamai Control Center for additional details

## API Integration

### EdgeGrid Authentication

The script uses HTTPie with EdgeGrid plugin for Akamai API calls:

```bash
http --auth-type edgegrid -a default: GET https://{host}/cps/v2/enrollments
```

### DigiCert API Integration

The script uses curl to submit CSRs to DigiCert MPKI:

```bash
curl https://demo.one.digicert.com/mpki/api/v1/certificate \
  -H "x-api-key: {API_KEY}" \
  -H "Content-Type: application/json" \
  -d @request.json
```

## Security Considerations

- **Credentials Storage**: EdgeGrid credentials stored in `~/.edgerc` with `600` permissions
- **Configuration Files**: Saved with `600` permissions (readable only by owner)
- **Backup Old Configs**: Previous configs are backed up with timestamp
- **API Keys**: Treat DigiCert API keys as secrets; rotate regularly
- **Log Files**: Contains URLs and identifiers; restrict access to log files

```yaml

kind: CronJob
metadata:
  name: akamai-cert-renewal
spec:
  schedule: "0 2 1 * *"  # 1st day of month at 2 AM
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: renewal
            image: akamai-cps:latest
            args: ["--renew", "example.com", "--auto"]
            volumeMounts:
            - name: config
              mountPath: /root/.akamai-cps

            secret:
              secretName: akamai-config
          - name: certs
            persistentVolumeClaim:
              claimName: akamai-certs-pvc
          restartPolicy: OnFailure
```

## Support and Issues

### Common Questions

**Q: How often should I renew certificates?**
A: Typically annually. You can schedule monthly renewal attempts via cron - the script will only proceed if Akamai has a pending renewal change.

**Q: Can I manage multiple domains?**
A: Yes! Run `--init` for each domain. Each gets its own enrollment and configuration.

**Q: Is the script production-ready?**
A: Yes, it includes error handling, polling, and confirmation steps. Use `--auto` mode with caution in production.

**Q: What if deployment fails mid-process?**
A: The script saves state throughout. You can inspect the JSON responses in the certificate storage directory and retry manually or run the renewal again.
